# 原型图（框架版）制作指南

目标：让技术小白**先看到界面大概长什么样**并给出反馈，再把确认后的版本以文本形式写进 PRD 供编码 agent 阅读。只做**核心主功能**界面，低保真、框架版——不要纠结配色、字体、像素，重点是"有哪些区域、怎么布局、点了会怎样"。

## 两步走

### 第 1 步：可视化预览（给用户看）

优先用**可交互的可视化预览**，因为小白看图比看 ASCII 更容易反应：

- 如果有 **visualize 的 mockup 能力**（`mcp__visualize__show_widget`，先调用 `read_me` 加载 mockup 模块），用它渲染一个低保真线框 mockup。
- 或者生成一个**简单的 HTML 预览**（灰底方块 + 占位文字的 wireframe 风格），让用户能在浏览器里看到。
- 预览只需表达布局和关键交互，用灰阶 + 占位符即可，别做成高保真成品。

### 第 2 步：确认 → 内嵌 ASCII 线框进 PRD

用 `AskUserQuestion` 弹框请用户二次确认：「这个布局对吗？」给出「就这样 / 要调整（说哪里）」之类的选项。

确认后，在最终 PRD 的"核心界面原型"一节里，用 **ASCII 线框图**表达同样的布局（纯文本，编码 agent 直接可读），并附一句交互说明。示例：

```
+------------------------------------------+
|  [Logo]                      [用户头像]   |
+------------------------------------------+
|  [ 输入框 ....................... ] [发送]|
|                                          |
|  对话区（消息气泡自上而下）                 |
|   ┌── 用户 ─────────────┐                 |
|   └─────────────────────┘                 |
|            ┌── AI ──────────────┐         |
|            └────────────────────┘         |
+------------------------------------------+
```
交互说明：在输入框输入后点「发送」→ 用户气泡立即出现在对话区底部 → AI 回复以流式追加在下方。

## 小提示

- 只画核心界面（通常 1-3 个）。次要页面（设置、关于等）不用画，列个名字即可。
- ASCII 图里用 `[按钮]`、`[输入框...]`、`{占位文字}` 等约定，清楚表达控件类型。
- 布局确认聚焦"信息层级和主操作在哪"，不要在这一步和用户纠结视觉细节。
