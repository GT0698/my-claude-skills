# 修订模式详规（用户已有笔记时）

## 何时进入
用户给了「已有笔记」（通常是飞书链接），可能还给了课件。两种子场景，可叠加：
- **A｜只整理**：没给新课件——把用户上课随手记的乱笔记理顺结构、规范排版。
- **B｜补充**：给了课件——把课件里用户遗漏的知识点补进已有笔记的对应位置。

## 铁律（不变）
帮他整理，不替他重写。**绝不删用户原话**，不改他的措辞。拿不准就保留。改的是"结构和补充"，不是"替换理解"。

## 范围红线：只动「课堂重点」
- 修订**只改「课堂重点」这一块**。**「课程资料」和「课后任务」不改、不删、不重排**——这两块是用户上课实时手记的，AI 不碰。
- **例外**：只有用户**明确要求**动这两块时才动；而且**动手前要再单独确认一次**（"你确定要改课程资料/课后任务吗？"），确认后只做用户要求的那点改动，同样不删原有内容。

## 步骤

### 1. 读
- 读用户已有笔记：用 `lark-doc`，读到**能定位 block** 的程度（`--detail with-ids` 或 full）——后面要做精准局部编辑，必须先拿到 block id。
- 场景 B 再读课件（飞书 / PPT / PDF）。

### 2. 整理（两种场景都做）
- 按排版规则规范：层级、编号（并列无序 / 递进数字、每节重起、无 1.1.1）、纲领句转**高亮块 callout**、段内加粗。
- 调结构：合并重复、补层级标题、调顺序、补逻辑衔接。
- 图：单图转结构化文字，保留原图并标"（图1）"。
- 以上都在**保留原话**的前提下做。

### 3. 补充（仅场景 B）
- 从课件提炼用户笔记里**缺**的知识点。
- **插到对应主题下**，不要另起一段堆在末尾——归到它该在的位置，融进结构。

### 4. 给新增/改动打临时标记
- 所有 **AI 新增或较大改动**的内容，用**黄色文字背景高亮**临时标出：`<span background-color="light-yellow">新增内容</span>`。
- 这跟正文纲领句用的"高亮块 callout"**不冲突**（一个是文字背景色、一个是整块），用户一眼能分清"临时新增标记" vs "正文纲领块"。
- **只标"新增和大改"**；纯格式规范（如把某行改成 H3）不用标，否则满屏高亮。

### 5. 改前确认 → 就地改
- **写回默认：就地改用户给的那篇原文档**（不新建副本）。飞书自带历史版本可回滚，是安全网。
- **动手前先确认**：把"改动清单"给用户过一眼——
  1. 调了哪些结构 / 合并了什么
  2. 从课件补了哪些点（场景 B）
  3. 明确申明：**没有删任何原文**

  用弹框或简短列表都行，等用户点头再动。
- 确认后，用 **`lark-doc` 的精准编辑**（`str_replace` / `block_insert_after` / `block_replace` 等局部指令）就地改，**绝不用 `overwrite` 整篇重写**——只有精准局部编辑才能真正保住用户原有的 block、措辞，以及笔记里可能有的评论、嵌入资源。

### 6. 收尾
- 告诉用户："黄色高亮的部分是本次新增/改动，核对无误后跟我说一声『清掉高亮』，我一键清除。"

## 实操要点（lark-cli，实测有效）
- **只重建课堂重点、又不碰资料/作业的稳妥做法**：先 `docs +fetch --detail with-ids` 拿到各 block id；把新的课堂重点正文（从 `<h2>` 开始，**不含** `# 课堂重点` 标题本身）用 `docs +update --command block_insert_after --block-id <课堂重点标题的 id>` 插进去；再用 `docs +update --command block_delete --block-id "id1,id2,…"`（逗号分隔可批量）删掉旧的课堂重点内容块。**全程不碰**课程资料 / 课后任务的 block，`overwrite` 一次都不用。
- **原话零丢失**：被删的旧块里的用户原话，必须已在新插入的正文里**逐字保留**（比如把两句题外话搬进"课堂随记"小节），再删旧块——删的是重复，不是原话。
- **`--content @文件` 要用相对路径**：lark-cli 不接受绝对路径，先 `cd` 到文件所在目录再用 `@文件名`（或用 `-` 从 stdin 读）。
- **删/改高风险操作要带 `--yes`**：如 `wiki +node-delete` 需 `--yes` 确认；仅在用户明确要求后使用。
- **已在副本审过内容时**：直接出**干净版**（不再上黄色临时标记），纲领句保留 callout 高亮块即可，省得用户再清一遍。
- **首次 / 大改动**：可先按用户意愿在**副本**上跑一遍验证，确认无误再对原文精修，避免动真格时出错丢原文。

### 7. 一键清除临时高亮（用户确认后）
用户核对无误、说"清掉高亮"时执行：
- 目标只清**临时新增标记**，即 `background-color="light-yellow"` 的**文字背景高亮 span**。
- **为什么能安全批量清**：正文纲领句用的是**高亮块 callout**（块级，不是 span），表头灰底用的是 `light-gray`——都不是 light-yellow 文字背景。所以"清所有 light-yellow 文字背景 span"**只会命中临时标记**，不会误伤正文样式。
- 做法：`docs +fetch` 读该文档 xml（`--detail with-ids`）→ 找出所有含 `<span background-color="light-yellow">…</span>` 的 block → 对每个 block 用 `docs +update --command block_replace` 写回**去掉该 span、保留内层文字与其他样式**的版本。
- 清完告诉用户："临时高亮已全部清除，正文的高亮块和表格样式不受影响。"

## 自检
- [ ] 只动了课堂重点；课程资料 / 课后任务没碰（除非用户明确要求且已二次确认）
- [ ] 用户原话一句没删
- [ ] 新增/大改都打了黄色高亮临时标记；纯格式规范没乱标
- [ ] 改前给了改动清单并拿到确认
- [ ] 用精准局部编辑就地改，没用 overwrite 覆盖整篇
- [ ] 课件补充的点归到了对应主题下，不是堆末尾
- [ ] （用户要求清高亮时）只清了 light-yellow 文字背景 span，callout 与表头灰底未受影响
