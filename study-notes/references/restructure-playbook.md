# 重组手法：课件 → 课堂笔记

笔记的用途是**日后复习速查**：忘了"某个结构下的某个知识点"时，看一眼就知道这个问题下要注意哪些细节。所以重点是把知识**按结构组织清楚**，而不是照抄课件。课件是按讲课顺序排的，还夹着标题页、过渡页、讲师介绍、致谢页、课程表——这些与知识无关的都删掉。

## 一、重组的基本动作

### 1. 按主题归堆，不按页码顺序
同一个概念可能散在好几页。把相关内容归到同一个主题标题下，形成"一个问题讲透"的块。

### 2. 定义 → 一问一答
概念定义用"X是什么？""为什么要X？"做小标题，下面给简明回答。最利于复习自测。

### 3. 删噪音、补逻辑
- **删**：标题页、过渡页、讲师介绍、课程表、致谢页、营销话术、与理解无关的重复。
- **补**：课件常缺"为什么"和"衔接"。适当补一句这个概念为什么重要、和上一个主题什么关系，让笔记连贯。补充要基于课件，别硬编事实。

## 二、结构与编号规则（重要，务必遵守）

### 层级
- 用标题层级组织：一级 → 二级 → 三级……**最多不超过 5 层**。最基础的一层是**正文**。
- 根据知识点的关系分配层级，不要为了凑层级而套壳。

### 标题的序号
- **优先单个位**：`一 / 二 / 三` 或 `1 / 2 / 3`。
- 需要时可以用两位：`1.1 / 1.2 / 1.3`。
- **序号的位数不要超过两位**——不要出现 `1.1.2`、`1.1.3`、`1.1.2.1` 这种。层级靠标题级别体现，不靠一长串点号。

### 正文里列点的标记：看知识点是"并列"还是"递进"
- **并列**（没有先后/递进关系）→ 用**无序标记**（`-`）。
- **递进 / 有层次 / 有次序**（步骤、流程、由浅入深）→ 用**数字顺序** `1. 2. 3.`。
- **数字在每个结构内部重新从 1 开始**。不要跨结构累加，也不要写成 `1.1.2 / 1.1.3` 这种嵌套编号；在某个结构内部就直接 `1 / 2 / 3 / 4`。

## 三、强调：高亮块 vs 加粗
- **纲领/结论句 → 用飞书「高亮块」(callout)**：一句结论性 / 纲领性 / 补充性的判断，单独放进一个**高亮块**（飞书 XML 的 `<callout>` 块）。**不要用文字背景色高亮**（那种在飞书里观感不好）。
- **加粗**：在一大段知识点内部，把**关键的部分**加粗，帮助扫读时抓重点。
- 分工：高亮块针对"独立的一句纲领/结论"，加粗针对"段落内部的关键词句"。
- 若这次产出的是 **markdown 草稿**而不是飞书文档，用 `>` 引用块代替高亮块（md 无法表达高亮块）。

## 四、表格 vs 层级结构
- **默认用层级结构**（标题+列表）来整理课件里的表格内容——它更适合速查、也更符合笔记的层级感。
- 只有当内容**确实是多维对照、用层级讲会很啰嗦**时，才用表格。例如：四次工业革命（时间/国家/时代/发明多列对照）、AI 产品五层结构（层/内容/角色/价值多列对照）——这类适合表格。
- 判断标准：能用"标题+缩进列表"讲清楚的，就别上表格；非得多列横向对照才清楚的，才用表格。

## 五、图片处理（分模式，务必区分）

课件里几乎全是图片/图表。处理方式取决于当前是哪种模式：

### 从零转笔记（课件直接转笔记）
- **不保留原图**。凡是能用文字/结构讲清楚的，一律转成文字结构表达。
- 课件本来就几乎全是图，保留图没有意义——目标是把图里的信息"文字化、结构化"，让笔记可读可搜。

### 修订模式（用户已有笔记里插了图）
- 用户原有笔记里可能插了一两张图。对**单张图先转成文字**（结构化整理）之后，**可以保留这张图**。
- 在转出的文字后面**标注它来自哪张图**，用简单的图片索引即可，如"（图1）""（见图3）"。给图片编个号，文字与图对应得上就行。

## 六、修订模式的保守规则（用户已有笔记时）
> 完整工作流（就地精准编辑、改前确认、新增打临时高亮、两种场景）见 `revise-mode.md`；这里是核心原则速记。

用户已经写了一部分，带着他自己的理解和措辞。铁律：**帮他整理，不替他重写。**
- ✅ 可以：调整章节顺序、补层级标题、合并重复、补逻辑衔接、把图转成结构化文字、按上面的编号/强调规则规范格式。
- 🚫 不要：删掉用户写的原话、把他的表述换成你的说法、大改措辞。
- **拿不准就保留**：不确定某段该不该动，保留原样。
- 改动较大时，写入飞书前先把"调整了哪些结构、新增了什么、没删任何原文"简述给用户确认。

## 七、产出前自检
- [ ] 三块齐全、顺序对（课堂重点 / 课程资料 / 课后任务）
- [ ] 课堂重点按主题重排，不是照抄课件页顺序
- [ ] 层级不超过 5 层；标题序号不超过两位（无 1.1.1）
- [ ] 并列用无序、递进用数字；数字在每个结构内部重新从 1 开始
- [ ] 结论/纲领/补充的一句话用了高亮；段内关键处用了加粗
- [ ] 能层级化的没硬上表格；只有复杂多维对照才用表格
- [ ] 图片：从零转笔记→不留图，全部文字化；修订模式→转文字后可留图并标"图N"
- [ ] 课程资料 / 课后任务只留格式占位，没替用户编内容
- [ ] （修订模式）用户原话一句没删
