# 技术选型思路（面向技术小白）

用户不懂技术，你要替他做选型并用大白话解释。核心原则：**选 Claude Code / Codex 最擅长、社区最成熟、踩坑最少的主流方案**——因为后面真正写代码的是编码 agent，选它熟悉的技术，成功率最高、小白后续维护也最省心。

选型三个维度：
- **产品大小**：一个页面的小工具，还是有账号体系的完整应用？
- **复杂度**：纯前端展示，还是要存数据、要登录、要调 AI？
- **时限**：想今天就跑起来，还是可以慢慢打磨？越急越要选"开箱即用"的方案。

## 常见产品类型 → 推荐栈（作为默认建议，不要照搬，按实际调整）

| 产品类型 | 推荐栈 | 一句话理由 |
|---------|-------|-----------|
| 落地页 / 简单展示工具 | 纯 HTML + Tailwind CSS（或 Next.js） | 无需后端，几分钟能出东西，部署到 Vercel/Netlify 免费 |
| 完整 Web 应用（有账号、有数据） | Next.js + Supabase（Postgres + 内置登录） | 一套搞定前端、数据库、认证，小白最省心 |
| 需要复杂后端逻辑 | 前端 Next.js/React + 后端 Node(Express) 或 Python(FastAPI) | 前后端分离，逻辑复杂时更清晰 |
| AI 产品（对话、生成、分析） | 前端 Next.js + 后端 Python(FastAPI) 或 Node + Claude API | Python/Node 生态对接大模型最顺，Claude API 能力强 |
| 手机 App | React Native 或 Flutter；轻量需求可先做 PWA（手机也能装的网页） | 一套代码双端；PWA 最快最省 |
| 桌面软件 | Tauri（轻）或 Electron（生态全） | 用网页技术做桌面应用 |
| 命令行工具 / 自动化脚本 | Python | 写得快、库最多，小白最容易看懂 |
| 数据处理 / 爬虫 | Python | 同上 |

## 常用配套（按需推荐）

- **数据库**：小项目用 SQLite / Supabase；正经项目用 Postgres（Supabase 自带）
- **登录认证**：Supabase Auth、Clerk、NextAuth——别让小白自己从零写登录
- **部署上线**：Vercel（Next.js 首选）、Netlify、Railway（带后端/数据库的）
- **样式**：Tailwind CSS（快、主流）
- **大模型**：Claude API（首选，能力强）；模型 ID 用最新的，如需可查 claude-api skill

## 给建议时的表达方式

- 每个推荐都配"**为什么选它**"的一句话大白话，别堆术语。
- 如果有更简单的替代能达到目的，主动告诉用户"其实还能更省事"。
- 最终用弹框让用户确认，给类似选项：「就按你推荐的来（推荐）」/「我想更简单/更省钱」/「我有指定的技术」。
